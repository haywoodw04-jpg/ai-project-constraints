# 安全规则

本文件约束本仓库和下游项目中的安全相关开发。当前仓库不包含登录、支付、数据库或用户数据；下游项目涉及这些能力时必须补全项目事实和验收方式。

## 一、登录安全

- 密码必须使用成熟哈希算法保存，禁止明文保存。
- 登录失败次数、验证码或限流策略必须明确。
- Session、Token、Cookie 的有效期、刷新策略和退出策略必须明确。

## 二、注册安全

- 注册入口必须具备频率限制。
- 用户提交的数据必须服务端校验。
- 邀请码、手机号、邮箱等唯一字段必须由数据库唯一约束兜底。

## 三、验证码防刷

- 验证码开关、必填场景、失败策略必须配置化。
- 禁止在前端只做验证码校验。
- 验证失败不得泄露内部校验细节。

## 四、管理后台权限

- 管理员接口必须验证登录态和角色。
- owner 权限不得被普通 admin 修改。
- 高危操作必须二次确认并记录操作日志。

## 五、用户数据权限

- 用户只能访问自己的数据。
- 服务端必须从登录态识别用户，不得直接相信前端传来的 `userId`。
- 用户查询订单、余额、积分、文件、任务时必须做归属校验。

## 六、水平越权防护

- 查询、更新、删除用户资源时必须同时校验资源 ID 和当前用户身份。
- 禁止只按资源 ID 查询后直接返回。
- 批量接口必须逐项校验权限或在查询条件中限制归属。

## 七、垂直越权防护

- 管理接口、运营接口、财务接口、系统配置接口必须区分角色。
- 禁止仅靠前端菜单隐藏实现权限。
- 权限失败必须返回统一错误，不暴露内部角色配置。

## 八、支付回调安全

- 支付回调必须验签。
- 金额必须以后端订单金额为准。
- 支付状态变更必须幂等。
- 支付成功后触发发货、会员、积分、余额变更必须使用事务或等效原子机制。

## 九、文件上传安全

- 必须限制类型、大小、扩展名、MIME、存储路径和访问权限。
- 上传文件不得直接放在可执行目录。
- 图片、文档等文件需要重新生成安全文件名。

## 十、敏感字段脱敏

不得在接口、日志、截图、文档中明文暴露：

- 密码、Token、Session、Cookie、API Key、密钥。
- 完整手机号、邮箱、证件号、银行卡号。
- 完整卡密、支付配置、回调签名密钥。

## 十一、日志安全

- 日志可以记录请求 ID、用户 ID、订单号、错误码。
- 日志不得记录密码、Token、密钥、完整卡密、完整支付回调敏感内容。
- 异常日志不得直接返回给用户。

## 十二、环境变量安全

- `.env` 不得提交到公开仓库。
- 文档只能写变量名和示例格式，不能写真实值。
- 生产密钥必须通过部署平台或进程环境注入。

## 十三、API 限流

- 登录、注册、验证码、支付、发货、导出、AI 生成等接口必须评估限流。
- 限流维度可以是 IP、用户 ID、设备、接口路径或业务 ID。
- 限流命中必须返回统一错误码。

## 十四、幂等和重放攻击防护

- 支付回调、扣费、发货、退款、加积分必须使用幂等键或唯一约束。
- 重要回调必须校验时间戳、签名或 nonce。
- 重放请求不得产生重复业务副作用。

## 十五、高危操作二次确认

以下操作必须二次确认并记录日志：

- 删除订单、用户、商品、文件、配置。
- 修改支付、权限、价格、库存、余额、积分、会员。
- 批量导入、批量导出、批量发货、批量退款。

## 十六、禁止事项

- 禁止相信前端传来的价格、积分、余额、权限、支付状态。
- 禁止普通 admin 修改 owner 权限。
- 禁止支付回调不验签。
- 禁止密码、Token、密钥、完整卡密写入日志。
- 禁止文件上传不限制类型、大小、路径和访问权限。
