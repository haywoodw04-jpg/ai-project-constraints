# API 契约

当前仓库不包含后端接口。本文件是下游项目的 API 约束模板；如果下游项目有后端接口，必须在开发接口前补全本文件。

## 一、统一返回格式

成功：

```json
{
  "success": true,
  "data": {},
  "error": null
}
```

失败：

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "ERROR_CODE",
    "message": "错误说明"
  }
}
```

禁止同一项目混用多套返回结构。

## 二、统一错误格式

- `error.code` 使用稳定字符串，不使用一会儿数字、一会儿字符串的混合格式。
- `error.message` 面向用户或调用方，不能包含堆栈、SQL、密钥、内部路径。
- 需要调试信息时写入服务端安全日志，不返回给前端。

## 三、错误码规范

- 错误码使用大写英文和下划线，例如 `ORDER_NOT_FOUND`。
- 同一错误码只能表达一个业务含义。
- 新增错误码必须补充本文件或项目内错误码表。

## 四、分页参数规范

请求参数统一使用：

```json
{
  "page": 1,
  "pageSize": 20
}
```

响应格式统一使用：

```json
{
  "items": [],
  "pagination": {
    "page": 1,
    "pageSize": 20,
    "total": 100,
    "totalPages": 5
  }
}
```

禁止在同一项目内混用 `page/pageSize`、`offset/limit`、`current/size`。

## 五、时间字段规范

- API 时间字段默认使用 ISO 8601 字符串或项目统一格式，具体格式由下游项目确认。
- 字段命名优先使用 `createdAt`、`updatedAt`、`deletedAt`。
- 禁止同一接口同时返回秒级时间戳和毫秒级时间戳而不说明。

## 六、金额字段规范

- 金额字段必须声明单位：分、元或最小货币单位。
- 支付、订单、余额、退款金额以后端订单记录为准，不能相信前端传入金额。
- 禁止同一字段一会儿返回字符串金额、一会儿返回数字金额。

## 七、空值规范

- 不存在的对象返回 `null`。
- 空列表返回 `[]`。
- 可选字符串为空时使用 `""` 还是 `null` 必须由项目统一确认。
- 禁止同一字段在不同接口中随意切换 `null`、空字符串、空对象。

## 八、敏感字段过滤规范

前台接口禁止返回：

- 密码、Token、Session、API Key、密钥。
- 后台备注、内部审计字段、完整支付配置。
- 完整卡密、完整证件号、完整手机号等敏感数据。
- 其他用户的私有数据。

## 九、管理端接口规范

- 管理端接口必须验证登录态和角色权限。
- owner、super admin 等最高权限不得被普通 admin 修改。
- 管理端查询必须分页，禁止无 LIMIT 返回大列表。
- 高危操作必须记录操作日志。

## 十、用户端接口规范

- 用户只能访问自己的数据。
- 用户端接口不得信任前端传来的用户 ID、角色、余额、积分、价格、库存、支付状态。
- 涉及订单、支付、发货、退款的接口必须具备幂等保护。

## 十一、文件上传接口规范

- 必须限制文件类型、大小、扩展名、MIME 和存储路径。
- 上传后访问权限必须明确。
- 禁止将用户上传文件直接作为可执行脚本或动态模板执行。

## 十二、幂等接口规范

- 支付回调、发货、扣费、加积分、退款、任务提交必须设计幂等键或唯一约束。
- 重复请求不得导致重复扣款、重复发货、重复加积分。
- 幂等处理结果必须可审计。

## 十三、禁止事项

- 禁止随意新增 endpoint 而不更新本文件。
- 禁止改变接口返回格式而不通知前后端。
- 禁止错误码一会儿数字一会儿字符串。
- 禁止分页一会儿 `page/pageSize`，一会儿 `offset/limit`。
- 禁止前台接口返回后台备注、密钥、完整卡密、Token、支付配置等敏感字段。
