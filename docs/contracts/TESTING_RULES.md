# 测试规则

当前仓库是 Markdown 文档仓库，没有业务测试体系。下游项目必须根据真实技术栈补充类型检查、lint、构建、单元测试、接口测试、数据库迁移测试和回归测试命令。

## 一、当前仓库必跑检查命令

```powershell
git status --short
rg --files docs AGENTS.md PROJECT.md
```

Markdown lint、链接检查、拼写检查：待人工确认。

## 二、类型检查

- TypeScript 项目必须运行类型检查命令，例如 `npm run typecheck` 或项目等价命令。
- Python 项目如配置了类型检查，应运行 `mypy`、`pyright` 或项目等价命令。
- 当前仓库无类型检查命令。

## 三、Lint 检查

- 前端或 Node 项目必须运行项目已有 lint 命令。
- Python 项目如配置了 ruff、flake8、pylint，应运行对应命令。
- 禁止为了通过 lint 删除有效规则或关闭大范围检查。

## 四、构建检查

- 有构建产物的项目必须运行构建命令。
- 构建失败不得用删除功能或绕过类型检查解决。
- 当前仓库无构建命令。

## 五、单元测试

- 新增核心纯函数、解析逻辑、金额计算、状态判断必须补充单元测试。
- 修复 bug 时优先添加能复现问题的测试。
- 禁止为了通过测试删除测试。

## 六、接口测试

- 新增或修改 API 必须验证成功、失败、权限不足、参数错误、空数据。
- 接口返回格式必须符合 `API_CONTRACT.md`。
- 支付、发货、扣费、加积分等接口必须测试重复请求。

## 七、数据库迁移测试

- 数据库迁移必须在测试库验证。
- 必须验证迁移前后关键查询和写入。
- 禁止测试代码连接生产数据库。

## 八、支付回调测试

- 必须测试验签失败、金额不一致、重复回调、订单不存在、订单已完成。
- 不得依赖真实支付扣款做自动化测试。
- 测试密钥必须是假密钥或测试环境密钥，不能写入仓库。

## 九、权限测试

- 必须测试未登录、普通用户、普通管理员、owner 或超级管理员的差异。
- 用户只能访问自己的数据。
- 管理员接口必须验证角色。

## 十、回归测试

- 修改共享工具、请求封装、状态机、权限、数据库 schema、支付、订单后必须扩大回归范围。
- 修改单点文案或样式可做最小回归。
- 回归范围必须在修改总结中说明。

## 十一、高危测试禁止事项

- 禁止测试代码连接生产数据库。
- 禁止测试代码执行无保护删除。
- 禁止测试代码包含真实密钥。
- 禁止测试代码依赖真实支付扣款。
- 禁止测试后不清理测试数据。
- 禁止为了通过测试而删除测试。

## 十二、编码后输出要求

每次修改总结必须写明：

- 实际运行的检查命令。
- 命令结果。
- 未运行的检查及原因。
- 仍需人工复核的风险点。
